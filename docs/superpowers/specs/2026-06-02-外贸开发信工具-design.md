# 外贸开发信工具 — 设计文档

> 日期：2026-06-02
> 公司：新乡金达反光制品有限公司 (Xinxiang Jinda Reflective Products Co., Ltd.)
> 状态：已确认，待生成实现计划

## 1. 目标与背景

为公司外贸团队（内部 5-10 人）做一个网页工具，半自动化"开发信"获客流程：

**核心价值**：同事点几下，就能找到一批欧美建筑/安全用品相关公司、抓到官网邮箱、用 AI 生成针对每家公司量身定制的英文开发信，导出 Excel 后粘贴到阿里巴巴小满邮箱发送。

**目标客户**：美国/欧洲建筑公司采购部门，以及高可视安全服 / 反光装备 / PPE 相关分销商，目标 1000 件以上批发订单。

## 2. 关键技术决策（含取舍说明）

| 决策点 | 选择 | 原因 |
|---|---|---|
| 联网搜索 | **Tavily API** | DeepSeek API 本身**不能联网**（联网搜索只在 DeepSeek 网页版，不在 API）。Tavily 专为 LLM 设计、每月 1000 次免费，内部用量基本零成本。 |
| 写开发信 | **DeepSeek API** | 中文团队熟悉、便宜，每封量身定制成本极低。 |
| 抓官网邮箱 | **直接 fetch 官网 HTML + 正则/DeepSeek 提取** | 免费，无需额外服务。 |
| 数据库 | **Neon (PostgreSQL)** | 免费额度，Serverless，与 Vercel 天然集成。 |
| 部署 | **Vercel 云端** | 免费，同事打开网址即用。 |
| 登录 | **不做登录**，仅 URL 轻量密码参数 | 内部小团队，避免 API 额度被外部乱用，但不引入账号系统复杂度。 |
| 邮件发送 | **不做**，用户自己粘贴到小满 | 明确需求；规避邮件送达/合规复杂度。 |
| 框架 | **Next.js 全栈** | 前后端一体，单一部署单元。 |

**外部 Key 总计 3 个**：DeepSeek（写信）+ Tavily（搜索）+ Neon（数据库连接串）。

## 3. 系统架构

```
浏览器（同事访问 Vercel 网址，URL 带轻量密码 ?key=xxx）
        │
        ▼
   Next.js 全栈应用（部署在 Vercel）
   ├─ 前端页面
   │    ├─ 搜索页：选地区 + 点分类卡片
   │    ├─ 公司列表页：表格 + 勾选 + 去重展示
   │    ├─ 开发信预览页：逐封查看/微调
   │    └─ 导出：下载 Excel
   └─ 后端 API 路由（app/api/*）
        ├─ /api/search   → 调 Tavily 搜公司 → 抓官网邮箱 → 去重入库
        ├─ /api/generate → 调 DeepSeek 为勾选公司生成开发信
        ├─ /api/export   → 生成 Excel 下载
        ├─ /api/companies→ 列表/状态读写
        └─ /api/settings → 读写公司卖点资料
        │
        ▼
   Neon (PostgreSQL)
```

## 4. 使用流程

1. **选地区**：下拉单选——美国 / 加拿大 / 英国 / 德国 / 澳大利亚 / 欧洲其他（默认美国）。
2. **点分类卡片**：7 个预设分类（见下），每个卡片背后是预写好的优化英文搜索词。点击即触发：Tavily 搜索 → 自动抓官网邮箱 → 与库内已有公司去重 → 入库。**全程不用打字。**
3. **公司列表**：表格展示（公司名 / 联系人 / 邮箱 / 地区 / 类型 / 来源 / 状态），自动去重。可勾选目标公司。无邮箱的标记"无邮箱"，不阻断。
4. **生成开发信**：勾选后点"生成开发信"，DeepSeek 为每家结合金达卖点 + 对方公司特点量身写英文信。
5. **预览/微调**：逐封查看，可手动编辑。
6. **导出 Excel**：格式与现有 `开发信客户邮箱_30.xlsx` 一致，增加"开发信主题"和"开发信正文"两列。
7. 用户复制粘贴到阿里巴巴小满邮箱发送。

## 5. 预设分类（7 个）

每个分类对应一条预写、可调优的英文 Tavily 搜索词模板（`{region}` 注入所选地区）：

| 卡片名称 | 抓取目标 | 搜索词方向（示意） |
|---|---|---|
| 高可视安全服 / 反光装备分销商 | high-visibility / reflective apparel 分销商 | `high visibility reflective safety apparel distributor {region}` |
| PPE / 劳保安全用品分销商 | 安全防护用品综合分销商 | `PPE safety supply distributor wholesale {region}` |
| 道路 / 工地 / 市政安全用品商 | 交通工地市政安全用品 | `road work traffic municipal safety equipment supplier {region}` |
| 工装 / 制服分销商 | workwear / uniform 进口分销商 | `workwear uniform distributor importer {region}` |
| 工业安全 / 安全设备商 | 工业安全设备供应商 | `industrial safety equipment supplier {region}` |
| 建筑总承包商采购部 | 直接找建筑公司采购（核心目标） | `construction general contractor procurement {region}` |
| 警用 / 安防装备分销商 | 警用市政安防装备 | `police municipal safety gear distributor {region}` |

> 搜索词为初始模板，后续可在代码常量中微调，无需改架构。

## 6. 数据库表（Neon / PostgreSQL）

**companies**
- `id` (pk)
- `company_name`
- `contact` (可空)
- `email` (可空，无邮箱时为 null)
- `region` (地区/国家)
- `customer_type` (来自所点分类)
- `source` (官网 URL 或来源说明)
- `search_category` (触发抓取的分类卡片)
- `status`（枚举：待联系 / 已生成 / 已导出）
- `created_at`
- 去重键：对 `email` 非空时按 email 去重；email 为空时按 `company_name + region` 去重。

**letters**
- `id` (pk)
- `company_id` (fk → companies)
- `subject`（英文主题行）
- `body`（英文正文）
- `created_at`
- 同一公司可多次重写（取最新）。

**settings**（单行配置，可在线编辑）
- `id` (pk，固定 1)
- `company_profile`（金达卖点资料，预填官网内容，见第 7 节）
- `updated_at`

## 7. 开发信生成（核心）

**输入给 DeepSeek 的素材：**
- 金达卖点（来自 settings.company_profile，预填如下）
- 对方公司：名称、客户类型、地区

**预填的金达卖点（company_profile 初始值）：**
- 13+ 年反光安全服生产经验，3000+㎡ 工厂，500+ 员工
- 年产 100 万+ 件安全背心，客户遍布 30+ 国家
- Costco、华为等大客户供应商，提供 OEM/ODM 定制
- 产品线：安全背心、防水反光夹克、安全T恤、安全帽、安全鞋、工装等
- 工厂直供，省去中间环节，1000 件以上批发价有竞争力
- 可按 ANSI/ISEA 107、CE EN ISO 20471 标准生产（注意：**目前无第三方认证证书，开发信不声称已认证，仅表述"可按标准生产 / can manufacture to standard"**）

**DeepSeek 输出：**
- `Subject`：吸引采购打开的英文主题行
- `Body`：专业 B2B 英文开发信，含称呼、价值主张（突出工厂直供 + 大客户背书 + 产能）、针对建筑采购痛点（高可视合规、批量价）、CTA（询价 / 索取产品册或报价单）、签名占位。

**风格**：简洁专业、不啰嗦、避免明显 AI 腔与夸大；不虚标认证。

## 8. 错误处理

- 搜不到 / 官网抓不到邮箱 → 公司仍入库，邮箱标"无邮箱"，不阻断后续。
- Tavily / DeepSeek API 调用失败 → 前端明确报错、可重试，已入库数据不丢。
- API 额度用尽 → 友好提示具体哪个服务额度用尽。
- URL 密码错误 → 返回简洁提示页，不暴露功能。

## 9. 不做的（YAGNI）

- ❌ 登录 / 账号系统（仅 URL 轻量密码）
- ❌ 邮件发送（用户用小满发）
- ❌ 多账号 / 权限区分 / 谁找的客户归属
- ❌ 自定义搜索词输入框（改为固定 7 分类）
- ❌ 阿里巴巴页面抓取（反爬，不做）

## 10. 测试策略

- **单元测试**：邮箱提取（正则）、去重逻辑、Excel 列映射、搜索词模板拼接。
- **集成测试**：API 路由对 Tavily/DeepSeek 做 mock，验证 search→入库、generate→落库、export→正确 Excel。
- **手动验证**：本地用真实 Key 跑一次完整流程（选地区→点1个分类→生成1封→导出），核对 Excel 与现有样本格式一致。
